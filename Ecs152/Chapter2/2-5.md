# Socket Programming

## Goal
Learn how to build client/server applications that communicate using sockets.

## What is a Socket?
A **socket** is the door between an application process and the end-to-end transport protocol.

![Socket Diagram](https://github.com/user-attachments/assets/cd015081-175f-42a9-9af1-e290a40fdea6)

Sockets are the **only API available** to application-layer code for accessing transport-layer services beneath it.

---

## Socket Programming Topics
- Socket abstraction  
- UDP sockets  
- TCP sockets  

If you want to directly access the Internet’s transport-layer services and send application-layer messages between distributed systems, you **must use sockets**. This applies across all operating systems.

---

## Transport Services

### Two Socket Types

| Protocol | Description |
|--------|------------|
| **UDP** | Unreliable datagram service |
| **TCP** | Reliable, byte stream–oriented service |

---

## Example Application Flow

1. Client reads a line of input from the keyboard  
2. Client sends the data to the server  
3. Server receives the data and converts it to uppercase  
4. Server sends the modified data back to the client  
5. Client receives and displays the result  

---

## UDP Socket Programming

### Key Characteristics
- No connection between client and server  
- No handshaking before sending data  
- Sender attaches destination IP address and port to each packet  
- Receiver extracts sender IP and port from incoming packet  
- Data may be **lost** or **arrive out of order**

### Application Viewpoint
UDP provides **unreliable transfer of datagrams (groups of bytes)** between client and server processes.

---

## Client/Server Interaction (UDP)
- Stateless communication  
- Each message is independent  
- No delivery guarantees  
- Faster, lower overhead than TCP

## UDP Client (Python)

```python
from socket import *

serverName = 'hostname'  # server name
serverPort = 12000

# Create UDP socket
clientSocket = socket(AF_INET, SOCK_DGRAM)

# Get user input
message = input('Input lowercase sentence: ')

# Send message to server
clientSocket.sendto(message.encode(), (serverName, serverPort))

# Receive response
modifiedMessage, serverAddress = clientSocket.recvfrom(2048)

print(modifiedMessage.decode())

# Close socket
clientSocket.close()
```

---

## UDP Server (Python)

```python
from socket import *

serverPort = 12000

# Create UDP socket
serverSocket = socket(AF_INET, SOCK_DGRAM)

# Bind socket to port
serverSocket.bind(("", serverPort))

print("The server is ready to receive")

while True:
    message, clientAddress = serverSocket.recvfrom(2048)

    # Process message
    modifiedMessage = message.decode().upper()

    # Send response back to client
    serverSocket.sendto(modifiedMessage.encode(), clientAddress)
```

## Socket Programming with TCP

### Key Concepts

- TCP is **connection-oriented**
- Client must contact server
- Server must:
  - Be running
  - Have a socket ready to accept connections

### Connection Flow

- Client creates TCP socket and specifies server IP + port  
- TCP connection is established  
- Server accepts connection and creates a **new socket** for that client  
- Each client gets its own dedicated connection socket  
- Server can handle multiple clients using different sockets  

### Application Viewpoint

TCP provides:
- Reliable delivery  
- In-order data  
- Byte-stream communication ("pipe")

## TCP Client (Python)

```python
from socket import *

serverName = 'hostname'
serverPort = 12000

# Create TCP socket
clientSocket = socket(AF_INET, SOCK_STREAM)

# Connect to server
clientSocket.connect((serverName, serverPort))

# Get input
sentence = input('Input lowercase sentence: ')

# Send data
clientSocket.send(sentence.encode())

# Receive response
modifiedSentence = clientSocket.recv(1024)

print('From Server:', modifiedSentence.decode())

# Close connection
clientSocket.close()
```

---

## TCP Server (Python)

```python
from socket import *

serverPort = 12000

# Create TCP socket
serverSocket = socket(AF_INET, SOCK_STREAM)

# Bind socket to port
serverSocket.bind(("", serverPort))

# Listen for connections
serverSocket.listen(1)

print("The server is ready to receive")

while True:
    # Accept connection
    connectionSocket, addr = serverSocket.accept()

    # Receive data
    sentence = connectionSocket.recv(1024).decode()

    # Process data
    capitalizedSentence = sentence.upper()

    # Send response
    connectionSocket.send(capitalizedSentence.encode())

    # Close connection
    connectionSocket.close()
```

## Summary: UDP vs TCP

| Feature     | UDP                  | TCP                         |
|------------|----------------------|-----------------------------|
| Connection | No                   | Yes                         |
| Reliability| Unreliable           | Reliable                    |
| Ordering   | Not guaranteed       | Guaranteed                  |
| Speed      | Faster               | Slower (overhead)           |
| Use Case   | Streaming, gaming    | Web, APIs, file transfer    |
