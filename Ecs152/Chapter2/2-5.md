## Socket Programming

Goal: learn how to build client/server applications that communicate using sockets.

Socket: door between an application process and the end-to-end transport protocol.

Sockets are the only API available to application-layer code for accessing the transport-layer services beneath it.

### Socket Programming:
- Socket abstraction  
- UDP sockets  
- TCP sockets  

If you want to directly access the Internet’s transport-layer services and send application-layer messages from one part of a distributed application to another, you need sockets. This is true for all operating systems.

Socket: door between the application process and the end-to-end transport protocol.

### Two Socket Types for Two Transport Services:
- UDP: unreliable datagram service  
- TCP: reliable, byte stream–oriented service  

---

### Application Example:

- Client reads a line of characters (data) from its keyboard and sends the data to the server  
- Server receives the data and converts the characters to uppercase  
- Server sends the modified data back to the client  
- Client receives the modified data and displays the line on its screen  

---


Socket Programming with UDP

UDP: no "connection" between client and server
    no handshaking before sending data
    sender explicitly attached IP destination address and port # to each packet
    receiver extracts sender IP address and port # from received packet

    UDP: transmitted data may be lost or received out-of-order

Application ViewPoint:
    UDP provides unreliable transfer of groups of bytes ("datagrams") between client and server processes

Client/Server Sockt Interaction: UDP

Example App:


UDP Client (Python)

from socket import * #include Python's socket library -> able to use Python's socket related methods

serverName = 'hostname' #server name and port number
serverPort = 12000

clientSocket = socket(AF_INET, SOCK_DGRAM) #create UDP socket for server

message = raw_input('input lowercase sentence:') #getting input

clientSocket.sendto(message.encode(), (serverName, serverPort)) #attach server name, port to message, send into socket. Sending applications layer message into the clients socket

modifiedMessage, serverAddress = clientSocket.recvfrom(2048) #client reads reply characters socket into string

print modifiedMessage.decode()
clientSocket.close() #closing socket

Socket Programming with TCP:
    TCP connections are client oriented
    Client must contact server
        server process must first be running
        server must have created socket (door) that welcomes client's contact
    Client contacts server by:
        creating TCP socket, specifying IP adress, port number of server process
        when client creates socket: client TCP establishes connection to server TCP
    When contacted by client, server TCP creates new socket for server process to communicate with that particular client
        allows server to talk with multiple clients
        source port numbers used to distinguish clients
    A new socket is created everytime a TCP client first contacts the TCP server and that this new socket is dedicated for communication with that specific client for the duration of that TCP connection
    Application Viewpoint:
        TCP provides reliable, in-order byte-stream transfer ("pipe") between client and server processes

Example App:

TCP Client: 
    from socket import *
    serverName = 'servername'
    serverPort = 12000
    clientSocket = socket(AF_INET, SOCK_STREAM) #TCP socket on client side creates
    clientSocket.connect((serverName, serverPort)) #before sending messages to the server, must connect to server
    sentence = raw_input('Input lowercase sentence:') #reading inpu
    clientSocket.send(sentence.encode()) #send input to client
    modifiedSentence = clientSocket.recv(1024) #client recieves reply 
    print('From Server:', modifiedSentence.decode()) #displays message
    clientSocket.close()

TCP Server:
    from socket import *
    serverPort = 12000
    serverSocket = socket(AF_INETT, SOCK_STREAM)
    serverSocket.bind((", serverPort)) #bind socket to port
    serverSocket.listen(1) #set socket into listening mode we can no recieve client connection requests
    print("The server is ready to receive")
    while True #each time through the loop we connect to a client, do the work, close the connection and loop back again and wait for another client connection request
        connectionSocket, addr = serverSocket.accept() #invoking the accept method. This is where the server will wait within the accept method until the client connects. When we return from the accept a new socket is created for us by the OS.

        sentence = connectionSocket.recv(1024).decode() #reads bytes from socket
        capitalizedSentence = sentence.upper()
        connectionSocket.send(capitalizedSentence.encode())

        connectionSocket.close()
