## The Domain Name System (DNS)

Part of the internet responsible for translating hostnames to IP addresses.

### Points of Interest:
- DNS structure and function  
- Resolving DNS queries  
- DNS record format  
- DNS protocol messages  

The role of DNS is to provide translation between names, services, and IP addresses.

---

### DNS:

- Distributed database implemented as a hierarchy of many name servers  
- Application-layer protocol: hosts and DNS servers communicate to resolve names (address/name translation)  
  - Core internet function, implemented at the application layer  
  - Complexity is at the network’s "edge"  

---

### Services and Structure

DNS provides multiple functions:
- Hostname-to-IP address translation  
- Host aliasing  
  - Canonical and alias names  
- Mail server aliasing  
- Load balancing for replicated web servers (many IPs per name)

#### Why not centralize DNS?
- Single point of failure  
- High traffic volume  
- Latency due to centralized location  
- Maintenance complexity  

#### Thinking About DNS:
- Massive distributed database:
  - Billions of records, each simple  
- Handles trillions of queries per day  
  - Many more reads than writes  
  - Performance is critical (milliseconds matter)

#### Organizationally and Physically Decentralized:
- Millions of organizations maintain their own records

### Root Name Servers (Layer 1)

- Last-resort servers when resolution fails  
- Critical to internet operation  
- DNSSEC provides security (authentication, integrity)  
- Managed by ICANN (Internet Corporation for Assigned Names and Numbers)  

---

### Top-Level Domain (TLD) Servers (Layer 2)

- Responsible for domains like `.com`, `.org`, `.net`, `.edu`, etc.  

---

### Authoritative DNS Servers (Layer 3)

- Organization’s own DNS servers  
- Provide hostname-to-IP mappings  
- Managed by the organization or a service provider  

---

### Local DNS Name Servers

- First point of contact for a host’s DNS query  
- Can:
  - Return cached results (may be outdated)  
  - Forward query into DNS hierarchy  

---

### DNS Name Resolution: Iterated Query

Example: host at `engineering.nyu.edu` wants IP for `gaia.cs.umass.edu`

Steps:
1. Host queries local DNS server (`dns.nyu.edu`)  
2. Local server queries root server  
3. Root server returns `.edu` TLD servers  
4. Local server queries TLD server  
5. TLD server returns authoritative server for `umass.edu`  
6. Local server queries authoritative server  
7. Authoritative server returns IP address

### Recursive Query Resolution

- The contacted server resolves the query fully  
- Puts more load on higher-level servers

### Caching DNS Information

- Servers cache mappings after resolution  
- Improves response time  
- Entries expire after TTL (Time To Live)  

Notes:
- Cached entries may be outdated  
- Updates propagate slowly until TTL expires  
- Best-effort system  

---

### DNS Records

DNS stores **resource records (RR)**:

Format:
(name, value, type, ttl)

Types:

- **Type A (Address Record):**
  - Name = hostname  
  - Value = IP address  

- **Type NS (Name Server Record):**
  - Name = domain  
  - Value = authoritative name server  

- **Type CNAME:**
  - Alias for a canonical name  
  - Example: `www.ibm.com` → `servereast.backup2.ibm.com`  

- **Type MX:**
  - Mail server for a domain  

---

### DNS Protocol Messages

DNS query and reply messages share the same format:

- **Header:**
  - ID: 16-bit identifier  
  - Flags: query/reply, recursion, authoritative  

- **Sections:**
  - Questions  
  - Answers  
  - Authority  
  - Additional

### Getting Your Info Into DNS

Example: new startup "Network Utopia"

1. Register `networkutopia.com` with a DNS registrar  
2. Provide authoritative name server info  
3. Registrar adds NS and A records to TLD servers  
4. Configure authoritative server:
   - A record for `www.networkutopia.com`  
   - MX record for `networkutopia.com`  

---

### DNS Security

**DDoS Attacks:**
- Attack root servers:
  - Not successful so far  
  - Mitigated via traffic filtering and caching  

- Attack TLD servers:
  - Potentially more dangerous
