## Connectionless Transport: UDP

UDP: User Datagram Protocol

UDP Sender/Receiver Actions  
UDP segment format  
Internet Checksum  

"no frills", "bare bones" Internet transport protocol  
"best effort" service — UDP segments may be lost and delivered out of order  

Connectionless:
- no handshaking between UDP sender and receiver  
- each UDP segment is handled independently of others  

### Why is there UDP?
- no connection establishment (which can add RTT delay)  
- simple: no connection state at sender or receiver  
- small header size  
- no congestion control  
  - UDP can send as fast as desired  
  - can still function in the face of congestion  

### UDP Use:
- streaming multimedia apps  
- DNS  
- SNMP  
- HTTP/3  

If reliable transfer is needed over UDP:
- add reliability at the application layer  
- add congestion control at the application layer  

---

### UDP: Transport Layer Actions

UDP Sender Actions:
- is passed an application-layer message  
- determines UDP segment header field values  
- creates UDP segment  
- passes segment to IP  

UDP Receiver Actions:
- receives segment from IP  
- checks UDP checksum header value  
- extracts application-layer message  
- demultiplexes message up to application via socket

### UDP Segment Header:

- only has 4 fields:  
  - source port number  
  - destination port number (used for multiplexing and demultiplexing)  
  - length (in bytes of UDP segment, including header)  
  - checksum  

---

### UDP Checksum:

Goal: detect errors (flipped bits) in transmitted segments  

Sender:
- treats contents of UDP segment (including UDP header fields and IP addresses) as a sequence of 16-bit integers  
- checksum: addition (one's complement sum) of segment contents  
- checksum value is placed into UDP checksum field  

Receiver:
- computes checksum of received segment  
- checks if computed checksum equals checksum field value:
  - not equal → error detected  
  - equal → no error detected (but errors may still exist in rare cases)  

---

### Summary: UDP

"no frills" protocol:
- segments may be lost or delivered out of order  
- best effort service: "send and hope for the best"  

UDP advantages:
- no setup/handshaking needed (no RTT incurred)  
- can function even when the network is congested  
- provides basic error detection (checksum)  

Additional functionality (reliability, congestion control) must be built on top of UDP at the application layer
